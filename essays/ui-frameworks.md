   UI frameworks are very important today for front end development. The standard for web page aesthetics has risen, and a slick looking web page is now expected by the general public. UI frameworks contain the tools necessary to meet this standard, and significantly reduce the amount of effort required. In most cases, not using one is practically reinventing the wheel, since you would be writing classes that already exist and are free to use. While learning how to use a framework takes some time, it would take vastly more effort to make a fully functional, responsive, good looking website with raw HTML and CSS.
   
   Semantic UI is a popular UI framework. Its purpose, along with other frameworks, is to make it easier for developers to make good looking web pages without having to write painstaking amounts of code for each and every component. Compared to competitors, one of Semantic UI's major selling points is its semantic syntax, where the framework's CSS classes are treated like words in a natural language. In other words, it is intended to be more intuitive to use than competing frameworks.
   
   At first, learning Semantic UI was somewhat frustrating. As someone who never before learned anything about front end development, getting accustomed to not only the various Semantic UI classes, but also HTML and CSS, took a little extra time. In my opinion, while Semantic UI's similarity to natural language is pretty neat, it was actually a little bit of a distraction from how classes work for an HTML beginner. That being said, I can see the feature being convenient for someone who is more familiar with HTML and CSS.
   
   While the pain of learning how to use a UI framework is temporary, the benefits of knowing how to create a fully functional, responsive, good looking website last a lifetime.
