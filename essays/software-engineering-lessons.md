   While we spent most of our time on developing for the web, the most important thing I learned from taking a software engineering course this semester was not how to develop a web application. Learning how to use HTML, CSS, JavaScript, React, and Meteor could certainly be very useful throughout a software engineer's career, but the most vital skill every software engineer must have is how to convert brainpower, time, and teamwork into a quality final product as efficiently as possible. This requires thoughtful project planning and structure, and is useful for developing any type of application.
  
## Configuration management
   Learning how to use Git is very important for software engineers. Git enables you to work on repositories of code with team members in a seamless way by tracking file changes on one's local machine and being able to sync them with a central server. Without knowing how to use Git (or some alternative form of configuration management) it would be vastly more difficult to create any type of medium to large scale application. Being able to know that the code you have is more or less in sync with everyone else is much better than sending files back and forth and hoping everything sticks together correctly in the end.
   
## Issue driven project management
   Issue driven project management is a useful way to structure projects. Under this model, each issue should take about three days to complete, and each team member should be working on one issue at a time. When issues are being created and assigned, this gives the team a vision of exactly what work will be required to reach a certain goal. This vision of the goal helps team member know what they should be working on at what time, and prevents inefficient overlap or rework of what others have completed. It can also prevent people from going down a particular rabbit hole and obsessing over some feature that isn't particularly important in the final product.
